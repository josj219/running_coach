## 1. 로컬 실행 기본 검증

- [ ] `pip install -r requirements.txt` 오류 없이 완료되는지 확인
- [ ] `streamlit run webapp/app.py` 명령으로 앱이 실행되는지 확인
- [ ] `http://localhost:8501` 접속 시 홈 화면이 정상 표시되는지 확인
- [ ] 사이드바에 5개 메뉴(주간 계획, 오늘 계획, 훈련 리뷰, 계획 조정, 주간 평가)가 표시되는지 확인

---

## 2. 환경변수 에러 처리

- [ ] `ANTHROPIC_API_KEY` 미설정 시 앱이 크래시하지 않고 명확한 안내 메시지를 표시하는지 확인
  - 기대 동작: "ANTHROPIC_API_KEY 환경변수가 설정되지 않았습니다." 또는 동등한 안내
  - 허용 불가: Python traceback이 그대로 노출되는 것
- [ ] `WORKSPACE_PATH` 미설정 시 기본값(`webapp/../`)으로 fallback 되는지 확인
- [ ] API 키가 올바르지 않을 때(invalid key) 사용자에게 명확한 에러 메시지를 보여주는지 확인

## 3. 핵심 기능 — 5개 스킬 작동 여부

### 3-1. 주간 계획 (pages/1_주간_계획.py)

- [ ] 페이지가 로드되는지 확인
- [ ] 이번 주 일정 입력 폼이 표시되는지 확인
- [ ] "계획 생성" 버튼 클릭 시 Claude API 호출이 시작되는지 확인
- [ ] 스피너/로딩 인디케이터가 표시되는지 확인
- [ ] 응답이 마크다운으로 정상 렌더링되는지 확인
- [ ] 생성된 계획이 `40-training-log/weekly/` 경로에 파일로 저장되는지 확인
- [ ] 저장된 파일명 형식이 올바른지 확인 (예: `2026-W23.md`)

### 3-2. 오늘 계획 (pages/2_오늘_계획.py)

- [ ] 페이지가 로드되는지 확인
- [ ] 오늘 날짜가 자동으로 표시되는지 확인
- [ ] 컨디션(1~10 또는 선택지) 입력 UI가 있는지 확인
- [ ] 날씨 입력 UI가 있는지 확인
- [ ] 주간 계획 파일을 자동으로 읽어 컨텍스트로 활용하는지 확인
- [ ] 생성된 당일 계획이 `40-training-log/daily/YYYY-MM-DD.md`에 저장되는지 확인

### 3-3. 훈련 리뷰 (pages/3_훈련_리뷰.py)

- [ ] 페이지가 로드되는지 확인
- [ ] 훈련 결과 입력 폼(거리, 시간, 평균 페이스, 심박수 등)이 있는지 확인
- [ ] 이미지 업로드 UI가 있는지 확인 (Garmin 스크린샷 등)
- [ ] 업로드된 이미지가 10MB 이하인 경우 정상 처리되는지 확인
- [ ] 리뷰 결과가 마크다운으로 정상 렌더링되는지 확인
- [ ] 당일 훈련 파일이 결과로 업데이트되는지 확인

### 3-4. 계획 조정 (pages/4_계획_조정.py)

- [ ] 페이지가 로드되는지 확인
- [ ] 조정 사유 입력 UI(피로도, 통증, 일정 변경 등)가 있는지 확인

- [ ] 조정된 계획이 주간 계획 파일에 반영되어 저장되는지 확인

### 3-5. 주간 평가 (pages/5_주간_평가.py)

- [ ] 페이지가 로드되는지 확인
- [ ] 이번 주 훈련 로그를 자동으로 불러오는지 확인

## 4. UX — 첫 화면 명확성

- [ ] 홈 화면에서 앱의 목적이 3초 안에 파악되는지 확인

  - 권장: "먼저 주간 계획을 생성하세요" 또는 현재 주의 상태 요약 표시

- [ ] 폼 입력 요소(텍스트 박스, 슬라이더 등)가 모바일에서 사용 가능한지 확인
- [ ] wide layout이 설정된 경우 모바일에서 가로 스크롤이 과도하게 발생하지 않는지 확인
  - 참고: Streamlit wide layout은 모바일에서 레이아웃이 좁아질 수 있음

## 6. 파일 저장 경로 검증

- [ ] `WORKSPACE_PATH` 환경변수가 실제 파일 저장에 사용되는지 확인
- [ ] 존재하지 않는 디렉토리에 저장 시 자동으로 생성되는지 확인 (mkdir -p 동작)
- [ ] 파일 저장 후 "파일이 저장되었습니다: {경로}" 안내가 표시되는지 확인
- [ ] Railway 환경(`WORKSPACE_PATH=/app/workspace`)에서 파일이 올바른 경로에 저장되는지 확인
  - 주의: Railway는 재시작 시 파일이 초기화됨 — 향후 퍼시스턴트 스토리지 연동 필요

## 8. 배포 전 반드시 수정해야 할 문제 목록

> 개발자가 구현 완료 후 이 섹션을 채워주세요.

- [ ] **[BLOCKER]** `ANTHROPIC_API_KEY` 미설정 시 앱 크래시 방지 처리 필요
- [ ] **[BLOCKER]** 파일 저장 경로가 환경변수를 제대로 참조하는지 검증 필요
- [ ] **[MAJOR]** 주간 계획 파일이 없을 때 오늘 계획/계획 조정/주간 평가 페이지의 fallback 처리 필요
- [ ] **[MAJOR]** Claude API 호출 타임아웃(긴 응답) 시 사용자에게 피드백 필요
- [ ] **[MINOR]** Railway 재시작 시 `/app/workspace` 파일 초기화 문제 — 현재 MVP에서는 허용, 향후 Railway Volume 또는 외부 스토리지 연동 권장
- [ ] **[MINOR]** 이미지 업로드 기능이 실제로 Claude API의 vision 기능을 활용하는지 확인 필요

## 9. 회귀 테스트 시나리오

다음 시나리오를 처음 사용자 관점에서 순서대로 테스트하세요:

1. 앱 실행 → 홈 화면 확인
2. 주간 계획 메뉴 → 이번 주 일정 입력 → 계획 생성 → 파일 저장 확인
3. 오늘 계획 메뉴 → 컨디션·날씨 입력 → 계획 생성 → 파일 저장 확인
4. 훈련 완료 후 → 훈련 리뷰 메뉴 → 결과 입력 → 리뷰 생성 → 파일 업데이트 확인
5. 피로 발생 → 계획 조정 메뉴 → 사유 입력 → 조정된 계획 확인
6. 주말 → 주간 평가 메뉴 → 평가 생성 → 리포트 저장 확인
