# /weekly-plan — 주간 훈련 계획 수립

> 목표·이력·컨디션·일정·날씨를 종합해 이번 주 훈련 계획을 수립하고 승인 후 저장한다.

---

## 핵심 원칙

- 하루 1회 훈련을 기본으로 한다. 추가 훈련은 사용자가 명시적으로 요청할 때만 반영.
- 실력 향상과 장기 성장을 최우선으로 한다.
- 부상 또는 통증 4/10 이상이면 훈련 강도보다 회복을 우선한다.
- 사용자가 승인하기 전까지 계획을 확정하지 않는다.
- 날씨·파일 조회가 실패해도 계획 생성을 멈추지 않고 폴백으로 진행한다.

---

## 프로젝트 경로 설정

```
PROFILE_FILE   = 20-knowledge-base/01_PROFILE.md
GOAL_FILE      = 20-knowledge-base/02_GOAL.md
WEEKLY_LOG_DIR = 40-training-log/weekly/
DAILY_LOG_DIR  = 40-training-log/daily/
HOLIDAYS_FILE  = .claude/skills/weekly-training-plan/references/holidays_{YEAR}.json
# YEAR는 현재 날짜 기준 연도로 동적 계산. 파일 없으면 LLM 폴백(한국 법정공휴일 기준, 음력 명절은 미확인 안내)
```

---

## 워크플로우

### Step 1: 컨텍스트 수집 [rag + api_mcp + prompt]

**1-a. 프로필 & 목표 읽기 [rag]**

Read `PROFILE_FILE`과 `GOAL_FILE`을 읽는다.
파일이 없으면 사용자에게 알리고 핵심 정보를 직접 입력받아 계속 진행한다.

**1-b. 최근 훈련 기록 읽기 [rag]**

`WEEKLY_LOG_DIR`에서 최근 2-3주치 파일을 읽는다. 파일 읽기 우선순위:
1. `YYYY-WNN_evaluation.md` — 주간 평가가 완료된 경우 우선 사용
2. `YYYY-WNN_plan.md`의 `## 훈련 기록` 섹션 — 평가 파일 없을 때 대체

파싱 대상 및 방법:
- **수행률**: `실행 세션 수 ÷ 계획 세션 수 × 100` (세션 수 기준). 값을 계산할 수 없으면 "수행률 불명"으로 표시.
- **훈련량(km)**: 요일별 거리 합산. 기록이 없으면 0으로 처리.
- **피로도/주간 평가**: `## 주간 평가` 섹션 자연어 요약.
- 파싱 실패 시 전체 파일 내용을 LLM이 자연어 요약으로 처리.
- 기록이 없으면 "훈련 이력 없음 (복귀 첫 주)"으로 처리.
- 수행률·km 모두 불명이면 Step 2에서 🟡 유지(보수적) 방향을 기본값으로 설정.

**1-c. 날씨 조회 [api_mcp]**

서울 기준 Open-Meteo API로 이번 주 날씨 예보를 가져온다.

```
GET https://api.open-meteo.com/v1/forecast?latitude=37.5665&longitude=126.9780&daily=precipitation_probability_max,temperature_2m_max,temperature_2m_min,windspeed_10m_max&timezone=Asia%2FSeoul&forecast_days=7
```

필요한 데이터: 일별 강수확률, 최고/최저 기온, 풍속.
- API 실패 또는 타임아웃 시: "날씨 정보 없음" 상태로 계속 진행. 계획에 "※ 날씨 미반영" 표시.

**1-d. 사용자 입력 수집 [prompt]**

단 한 번의 질문으로 필요한 정보를 모두 받는다. AskUserQuestion을 사용해 질문한다:

```json
{
  "questions": [
    {
      "question": "이번 주 일정과 컨디션을 알려주세요. (예: 수요일 회식, 금요일 야근, 무릎 약간 뻐근)",
      "header": "이번 주 상황",
      "options": [
        {
          "label": "특별한 제약 없음 / 컨디션 양호",
          "description": "일정 제약 없고 통증도 없어요. 정상 훈련 진행."
        },
        {
          "label": "일정 제약 있음 (직접 입력)",
          "description": "못 하는 날/시간, 컨디션, 통증 여부를 자유롭게 적어주세요."
        },
        {
          "label": "컨디션 저하 / 부상 회복 중",
          "description": "통증이 있거나 몸 상태가 좋지 않아요. 회복 중심으로 계획."
        }
      ],
      "multiSelect": false
    }
  ]
}
```

사용자가 "직접 입력(Other)"을 선택하면 자유 입력을 그대로 파싱한다.
- 통증 수준이 모호하면 추가 확인: "통증 정도를 1-10으로 말해줄 수 있어?"
- 통증 4/10 이상이면 Step 2에서 회복 방향으로 강제 설정.

---

### Step 2: 훈련 부하 분석 [prompt + script]

수집된 정보를 바탕으로 다음을 판단한다.

**훈련 방향 결정 (우선순위 순서대로 적용):**

| 우선순위 | 조건 | 방향 |
|---------|------|------|
| 1 (최우선) | 통증 4/10 이상 또는 부상 중 | 🔴 회복 우선 — 강도/볼륨 50% 이하 |
| 2 | 통증 1-3/10 또는 최근 2주+ 비훈련 | 🟡 기초 복귀 — 강도/볼륨 60-70% |
| 3 | 수행률 불명 또는 파싱 실패 | 🟡 유지 — 보수적 기본값 |
| 4 | 직전 주 수행률 50% 미만, 통증 없음 | 🟡 유지 — 직전 주와 동일 수준 |
| 5 | 직전 주 수행률 80% 이상, 통증 없음 | 🟢 점진 증가 — 직전 주 대비 +10% 이내 |

**우선순위 규칙:** 통증이 1/10이라도 존재하면 수행률 조건과 무관하게 🟡 이하로 고정. 조건이 겹칠 경우 번호가 낮은(우선순위 높은) 방향을 선택.
**통증 수치 모호 시:** "아파요", "약간" 등 자연어 입력은 반드시 0-10 재질문으로 수치화 후 판단. 당일 최고값 기준 적용.

**요일별 헬스장 / 야외 가능 여부 계산:**

```python
# 로직 (Python 또는 LLM 처리)
# 1. 이번 주 월~일 날짜 계산 (ISO 주차 기준)
# 2. holidays_2026.json 읽기
# 3. 공휴일 + 주말 = 헬스장 불가
# 4. 사용자가 입력한 불가 일정 반영
# 5. 날씨 강수확률 70% 이상 = 야외 러닝 주의 표시
```

**주간 훈련량 상한:**
- 🔴 회복: 최대 2-3세션, 달리기 볼륨 0-10km
- 🟡 기초 복귀: 최대 3-4세션, 달리기 볼륨 직전 주 60-70%
- 🟢 점진 증가: 최대 4-5세션, 달리기 볼륨 직전 주 +10% 이내

---

### Step 3: 주간 계획 초안 생성 [generate]

아래 형식으로 주간 계획 초안을 생성한다. 계획은 **사용자에게 텍스트로 출력**한다.

**출력 형식:**

```
## 주간 훈련 계획 — YYYY년 NN주차 (M/D ~ M/D)

### 최종 목표 맥락
| 목표 | 일정 | 남은 기간 | 필요 페이스 |
|------|------|----------|------------|
| {목표} | {일정} | {N}주 | km당 {페이스} |

### 지난 주 평가 ({W번호}: M/D ~ M/D)
- 훈련량: {N}km / {N}세션 — {핵심 평가 한 줄}

### 이번 주 방향
> {한 줄 방향}

### 요일별 계획
| 날짜 | 요일 | 구분 | 훈련 | 소요시간 |
|------|------|------|------|----------|
| ...  |      |      |      |          |

### 이번 주 목표 지표
| 지표 | 목표 |
|------|------|
| 총 훈련 세션 | N회 |
| 달리기 거리 | Nkm |
| 완료 기준 | ... |

### 주의사항
- ...
```

계획 생성 시 반드시 포함:
- 공휴일/주말에 헬스장 세션 배정하지 않음
- 훈련 목적 명시 ("왜 이 훈련인지" 한 줄)
- 부상 또는 회복 중이면 계획 상단에 ⚠️ 경고 표시
- 날씨 정보 없으면 야외 러닝 세션에 "※ 날씨 미확인" 표시

### Step 4: 사용자 검토 및 승인 [review]

초안 출력 후 AskUserQuestion으로 승인을 받는다:

```json
{
  "questions": [
    {
      "question": "이 계획으로 진행할까요?",
      "header": "계획 승인",
      "options": [
        {
          "label": "승인 — 파일로 저장",
          "description": "40-training-log/weekly/ 에 자동 저장합니다."
        },
        {
          "label": "수정 요청",
          "description": "바꾸고 싶은 부분을 알려주세요. 수정 후 다시 보여드립니다."
        },
        {
          "label": "저장 없이 참고만",
          "description": "계획은 대화에만 남기고 파일로 저장하지 않습니다."
        }
      ],
      "multiSelect": false
    }
  ]
}
```

"수정 요청" 선택 시: 수정 내용을 반영해 Step 3를 재실행한 뒤 다시 승인 요청.

### Step 5: 파일 저장 [script]

승인 시 `40-training-log/weekly/YYYY-WNN_plan.md`에 저장한다.

파일명 계산:
```python
import datetime
today = datetime.date.today()
iso = today.isocalendar()
filename = f"{iso[0]}-W{iso[1]:02d}_plan.md"
# 예: 2026-W23_plan.md
```

저장 파일 구조:
```markdown
# 주간 훈련 계획 — YYYY년 NN주차 (M/D ~ M/D)

## 최종 목표 맥락
...

## 지난 주 평가
...

## 이번 주 방향
...

## 요일별 계획
...

## 훈련 기록 (실행 후 기입)
| 날짜 | 실행 여부 | 비고 |
|------|----------|------|
| M/D  | —        | —    |
...

## 주간 평가 (주 종료 후 작성)
_미작성_
```

저장 완료 후 파일 경로를 사용자에게 안내한다.

같은 파일이 이미 존재하면 덮어쓰기 전 반드시 확인한다:
- 기존 `## 훈련 기록` 섹션에 `— | —` 이외의 값(✅/❌/⚠️)이 하나라도 있으면 **덮어쓰기 차단**.
- "이미 기록된 훈련 데이터가 있습니다. 계획 섹션만 업데이트할까요?" 로 안내.
- 사용자 동의 시: 기존 `## 훈련 기록` 섹션은 보존하고 나머지 섹션만 교체.

| 상황 | 처리 |
|------|------|
| 프로필 파일 없음 | 핵심 정보 직접 입력 받고 계속 진행 |
