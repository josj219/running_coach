# /plan-adjustment — 훈련 계획 조정

> workout-review의 진단을 처방으로 전환한다. 이번 주 남은 세션만 조정하고, 전면 재계획이 필요하면 `/weekly-plan`으로 위임한다.

---

## 핵심 원칙

- 계획은 가능한 유지한다. 불필요한 변경은 하지 않는다.
- 단기 성과보다 장기 성장을 우선한다.
- 통증 또는 부상 위험이 있으면 회복을 우선한다.
- 수정 이유를 반드시 설명한다. 근거 없는 변경은 없다.
- 이미 수행된 세션(오늘 이전 날짜, ✅/⚠️/❌ 기입된 행)은 절대 수정하지 않는다.
- `## 이번 주 방향` 섹션은 건드리지 않는다. 방향 변경이 필요한 수준이면 `/weekly-plan` 위임.

---

## 프로젝트 경로

```
WEEKLY_LOG_DIR = 40-training-log/weekly/
DAILY_LOG_DIR  = 40-training-log/daily/
PROFILE_FILE   = 20-knowledge-base/01_PROFILE.md
```

---

## 워크플로우

### Step 1: 컨텍스트 신규 로드 [rag]

반드시 실행 시점에 파일을 새로 읽는다. 캐시된 상태를 사용하지 않는다.

```python
import datetime
today = datetime.date.today()
iso = today.isocalendar()
weekly_file = f"{iso[0]}-W{iso[1]:02d}_plan.md"
```

읽어야 할 파일:
1. `WEEKLY_LOG_DIR/YYYY-WNN_plan.md` → `## 요일별 계획` 표 + `## 훈련 기록` 표 (이미 수행된 날 파악)
2. `DAILY_LOG_DIR/YYYY-MM-DD.md` (오늘 포함 최근 7일) → `## 훈련 리뷰` 섹션에서 회복 필요도, 달성률, 통증 이력 추출
3. `PROFILE_FILE` → 부상 이력, 고질적 통증 부위 (부위별 민감도 적용)

**수정 대상 날짜 계산:** 오늘 포함 이번 주 끝(일요일)까지의 날짜 중, `## 훈련 기록` 표에서 `— | —` 상태인 날짜만 수정 가능.

---

### Step 2: 현재 상태 입력 수집 [prompt]

workout-review가 오늘 이미 실행됐으면 daily log에서 읽어 재사용한다. 그 외 상태 변화만 추가로 묻는다.

AskUserQuestion을 사용해 1회 질문:

```json
{
  "questions": [
    {
      "question": "지금 컨디션과 이번 주 일정 변경이 있으면 알려주세요.",
      "header": "현재 상태",
      "options": [
        {
          "label": "컨디션 이상 없음 / 일정 변경도 없음",
          "description": "workout-review 결과만 반영해서 조정 여부를 판단합니다."
        },
        {
          "label": "피로/통증 있음 (직접 입력)",
          "description": "피로도(1-10), 통증 부위와 수준(1-10), 수면 시간을 알려주세요."
        },
        {
          "label": "일정 변경 있음 (직접 입력)",
          "description": "변경된 날짜와 내용을 알려주세요. (예: 목요일 출장, 토요일 행사)"
        }
      ],
      "multiSelect": false
    }
  ]
}
```

통증 수준이 자연어로 입력되면 0-10 재질문으로 수치화한다.

---

### Step 3: 조정 필요도 판단 [prompt]

수집된 모든 데이터(workout-review 이력 + 새 입력)로 판단한다.

**판단 기준 테이블 (우선순위 순서대로 적용):**

| 우선순위 | 조건 | 결정 | 조치 범위 |
|---------|------|------|-----------|
| 1 | 통증 4/10 이상 | 🔴 강제 조정 | 해당일 회복 루틴으로 교체, 이후 일정도 검토 |
| 2 | 일정 변경 (헬스장 불가일 + 헬스장 세션 배정됨) | 🔴 강제 조정 | 해당일 세션 교체 또는 재배치 |
| 3 | 통증 1-3/10 **2회 연속** 이상 (7일 이력 기준) | 🟡 선택 조정 | 해당 부위 자극 없는 세션으로 교체 |
| 4 | 피로도 7/10 이상 또는 수면 5시간 이하 | 🟡 선택 조정 | 강도 10-20% 하향 또는 세션 단축 |
| 5 | workout-review 달성률 60% 미만 | 🟡 선택 조정 | 당일 강도 하향 또는 볼륨 조정 |
| 6 | 일정 변경만 있음 (훈련 가능한 날로 이동 가능) | 🟡 선택 조정 | 훈련 재배치 (부하 패턴 유지 여부 검토) |
| 7 | 모든 조건 해당 없음 | 🟢 유지 | 수정 없음 |

**복합 조건:** 여러 조건 해당 시 번호가 낮은(우선순위 높은) 방향을 따른다.

**전면 재계획 트리거:** 🔴 조건이 남은 3개 이상의 세션에 영향을 미치면 직접 수정 대신 `/weekly-plan` 재실행을 권유하고 종료한다.

---

### Step 4: 조정 초안 생성 [generate]

조정이 필요한 날짜의 세션만 생성한다. 유지하는 날은 출력에 포함하지 않는다.

**출력 형식:**

```
## 훈련 계획 조정 — M/D 기준

### 판단 근거
- 회복 필요도: {workout-review에서 인용 또는 현재 입력}
- 부상 이력 참조: {PROFILE_FILE에서 읽은 고질적 통증 부위}
- 통증 누적 신호: {7일 이력 기준}

### 변경 세션

**M/D (요일)**
- 변경 전: {원래 훈련 내용, 소요시간}
- 변경 후: {수정된 훈련 내용, 소요시간}
- 변경 이유: {구체적 근거 — "피로도 7/10, 직전 세션 달성률 65%"}
- 주간 목표 영향: {이번 주 총 훈련량 Nkm → Nkm, 허용 범위 내/외}

[추가 변경 세션이 있으면 동일 형식으로 반복]

### 유지 세션
- {변경 없는 날짜 목록}

### 주간 계획 전면 수정 여부
- {불필요 / 필요 — 사유}
```

---

### Step 5: 사용자 승인 [review]

AskUserQuestion으로 확인받는다. 날짜별 개별 수락/거절이 가능함을 안내한다.

```json
{
  "questions": [
    {
      "question": "조정 내용을 확인해주세요. 전체 또는 일부만 수락할 수 있습니다.",
      "header": "계획 승인",
      "options": [
        {
          "label": "전체 수락 — 파일 저장",
          "description": "모든 변경 내용을 주간 계획 파일에 반영합니다."
        },
        {
          "label": "일부만 수락",
          "description": "어느 날짜의 변경을 수락하고 어느 날짜는 유지할지 알려주세요."
        },
        {
          "label": "계획 유지 — 변경 없음",
          "description": "현재 계획을 그대로 유지합니다."
        }
      ],
      "multiSelect": false
    }
  ]
}
```

### Step 6: 파일 수정 [script]

승인된 날짜의 세션만 수정한다.

**수정 전 필수 체크:**
- 해당 날짜가 오늘 이후인지 확인 (오늘 이전 날짜 수정 금지)
- `## 훈련 기록` 표에서 해당 날짜 행의 `실행 여부` 컬럼이 `— | —` 상태인지 확인 (✅/⚠️/❌ 기입된 행 수정 금지)
- 수정 대상이 `## 요일별 계획` 표 내 행인지 확인 (`## 훈련 기록` 표와 혼동 금지)

**수정 방법:**
- Edit 도구 사용. `old_string`에 날짜(M/D)를 반드시 포함해 행 고유성 확보
- 수정 후 파일 재읽어 반영 확인

**변경 이력 기록:**
`## 주간 평가` 섹션 또는 `## 조정 이력` 섹션(없으면 추가)에 변경 사유를 append한다:
```
- M/D 조정: {원래 세션} → {변경 세션} / 사유: {이유} / 시점: {오늘 날짜}
```

수정 완료 후 사용자에게 변경된 날짜 목록과 파일 경로를 안내한다.

## 폴백 처리

| 상황 | 처리 |
|------|------|
| 주간 계획 파일 없음 | `/weekly-plan` 먼저 실행 안내 후 종료 |
| 수정 가능한 날짜 없음 (이번 주 모두 완료 또는 오늘이 마지막 날) | "이번 주 남은 훈련이 없습니다" 안내 |
| 전면 재계획 필요 수준 | 직접 수정 없이 `/weekly-plan` 재실행 권유 |
| 파일 수정 실패 | 결과를 대화에 유지, 수동 수정 방법 안내 |
| workout-review 결과 없음 | 사용자 직접 입력으로만 판단, "이력 없음" 명시 |
