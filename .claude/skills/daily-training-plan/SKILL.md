# /today-plan — 당일 훈련 계획 & 결과 기록

> 주간 계획을 기반으로 오늘의 상세 훈련 계획을 생성하거나, 완료한 훈련 결과를 기록하고 주간 계획에 반영한다.

---

## 진입점 판단

사용자 입력을 보고 즉시 판단한다:

| 상황 | 실행 모드 |
|------|----------|
| "오늘 훈련 계획", "오늘 뭐 해", "당일 계획" | **[계획 모드]** Step 1부터 실행 |
| "훈련 완료", "결과 입력", "오늘 뛰었어" | **[기록 모드]** Step R1부터 실행 |
| 판단 불명확 | AskUserQuestion으로 모드 선택 |

## [계획 모드] 당일 훈련 계획 생성

### Step 1: 오늘의 주간 계획 확인 [rag]

오늘 날짜를 기준으로 이번 주 ISO 주차를 계산하고 `YYYY-WNN_plan.md` 파일을 읽는다.

```python
import datetime
today = datetime.date.today()
iso = today.isocalendar()
filename = f"{iso[0]}-W{iso[1]:02d}_plan.md"
```

파일에서 오늘 날짜에 해당하는 행을 찾아 계획된 훈련 내용을 추출한다.
- 파일이 없거나 오늘 날짜 행이 없으면: "주간 계획이 없습니다. `/weekly-plan`으로 먼저 주간 계획을 세워주세요." 안내 후 종료.
- 오늘이 휴식일로 계획된 경우: 사용자에게 알리고 계획 모드 종료.

### Step 2: 사용자 입력 수집 [prompt]

AskUserQuestion으로 두 가지를 한 번에 확인한다:

```json
{
  "questions": [
    {
      "question": "오늘 훈련 가능한가요? 컨디션도 알려주세요.",
      "header": "오늘 상태",
      "options": [
        {
          "label": "가능 / 컨디션 좋음",
          "description": "계획대로 진행합니다."
        },
        {
          "label": "가능 / 컨디션 보통",
          "description": "강도를 약간 낮춰서 진행합니다."
        },
        {
          "label": "가능 / 컨디션 저하 또는 통증",
          "description": "통증 위치와 강도(1-10)를 직접 입력해주세요."
        },
        {
          "label": "오늘 훈련 불가",
          "description": "이유를 알려주시면 주간 계획에 반영합니다."
        }
      ],
      "multiSelect": false
    }
  ]
}
```

"컨디션 저하 또는 통증" 선택 시: 통증 위치와 수준(1-10)을 추가로 확인한다.
통증 4/10 이상이면 Step 3에서 회복 세션으로 강제 변경.

"훈련 불가" 선택 시: Step 4(주간 계획 반영)로 바로 이동.

### Step 3: 날씨 확인 [api_mcp]

오늘 날씨를 Open-Meteo API로 조회한다. 야외 세션이 계획된 경우만 실행.

```
GET https://api.open-meteo.com/v1/forecast?latitude=37.5665&longitude=126.9780&daily=precipitation_probability_max,temperature_2m_max,temperature_2m_min&timezone=Asia%2FSeoul&forecast_days=1
```

- 강수확률 70% 이상: 야외 → 실내 대체 세션 제안
- API 실패: "날씨 미확인"으로 계속 진행

### Step 4: 당일 계획 결정 [prompt]

수집된 정보를 바탕으로 계획 변경 여부를 판단한다.

**변경 판단 기준:**

| 조건 | 조치 |
|------|------|
| 컨디션 좋음, 날씨 양호 | 주간 계획 그대로 실행 |
| 컨디션 보통 | 세트 수 또는 강도 10-20% 하향 |
| 통증 1-3/10 | 해당 부위 자극 없는 세션으로 교체 |
| 통증 4/10 이상 | 회복 루틴(스트레칭 + 가동성)으로 교체 |
| 야외 계획 + 강수 70%+ | 실내 대체 세션으로 교체 |
| 훈련 불가 | 해당 날 취소 후 주간 계획 조정 검토 |

주간 계획의 전체 구조(목표, 방향)는 유지한다. 오늘 세션만 조정.

### Step 5: 상세 훈련 계획 출력 [generate]

아래 형식으로 상세 계획을 출력한다. 주간 계획과 다른 경우 변경 이유를 명시.

---

**출력 형식:**

```
## 오늘의 훈련 계획 — M/D (요일)

### 훈련 개요
- 종류: {훈련 유형}
- 장소: {실내/야외, 헬스장/집}
- 소요시간: {N분}
- 목적: {이 훈련을 하는 이유 — 한 줄}

[변경된 경우만]
> ⚠️ 주간 계획 변경: {원래 계획} → {오늘 계획}
> 이유: {컨디션/날씨/부상}

---

### 워밍업 ({N분})
| 운동 | 세트 | 횟수/시간 | 유의점 |
|------|------|-----------|--------|
| ...  |      |           |        |

---

### 메인 훈련 ({N분})
| 운동 | 세트 × 횟수 | 강도/페이스 | 폼 포인트 |
|------|------------|------------|----------|
| ...  |            |            |          |

[러닝 세션인 경우 추가]
- 목표 페이스: km당 {페이스}  (※ 헬스장 트레드밀이면 "{속도} km/h ({페이스}/km)"로 표기 — 예: 9.0 km/h (6:40/km))
- 목표 심박: {N}-{N} bpm
- 목표 케이던스: {N} spm
- 추천 코스: {코스 설명}

---

### 쿨다운 ({N분})
| 운동 | 시간 | 유의점 |
|------|------|--------|
| ...  |      |        |

---

### 오늘 핵심 포커스
- {폼 교정 포인트 또는 집중해야 할 것 1-2가지}

### 완료 기준
- {이 훈련을 "성공"으로 볼 수 있는 기준}
```

### Step 6: 주간 계획 파일 업데이트 [script]

오늘 계획이 주간 계획과 다른 경우, 또는 훈련 불가인 경우에만 실행.

**수정 전 반드시 diff를 사용자에게 보여주고 확인받는다:**

```
📝 주간 계획 변경 예정:
- 파일: 40-training-log/weekly/YYYY-WNN_plan.md
- 변경 전: | M/D | 요일 | 구분 | {원래 계획} | {소요시간} |
- 변경 후: | M/D | 요일 | 구분 | {변경 계획} ({사유}) | {소요시간} |
반영할까요? [예 / 아니오]
```

확인 후 Edit 도구로 해당 날짜 행 전체를 `old_string`으로 지정해 수정한다.
- `old_string`에 날짜(M/D)를 반드시 포함해 다른 행과 구별한다.
- 수정 완료 후 파일을 재읽어 변경 내용이 정확히 반영됐는지 검증한다.

## [기록 모드] 훈련 결과 기록

### Step R0: 오늘의 주간 계획 확인 [rag]

계획 모드의 Step 1과 동일하게 오늘 날짜의 주간 계획을 읽는다.

```python
import datetime
today = datetime.date.today()
iso = today.isocalendar()
filename = f"{iso[0]}-W{iso[1]:02d}_plan.md"
```

`WEEKLY_LOG_DIR`에서 파일을 찾아 오늘 날짜 세션 헤더(`### M/D (요일)`)를 기준으로 계획된 훈련 내용을 추출한다.
- 파일이 없으면: 비교 없이 결과만 기록하고 계속 진행 (분석에 "주간 계획 없음" 표시).
- 오늘 세션이 없으면: "오늘은 계획상 훈련 없는 날이었습니다" 안내 후 결과 기록만 진행.

### Step R1: 결과 입력 수집 [prompt]

사용자가 자유롭게 입력한 내용을 파싱한다. 구조화된 입력을 강제하지 않는다.

필요하면 누락된 정보를 추가 질문한다 (최대 2개):

```json
{
  "questions": [
    {
      "question": "훈련 결과를 알려주세요. 거리, 시간, 페이스, 컨디션 등 아는 것만 말해도 됩니다.",
      "header": "훈련 결과",
      "options": [
        {
          "label": "직접 입력",
          "description": "예: 5km 28분, 심박 145, 컨디션 양호, 왼쪽 발목 약간 뻐근"
        },
        {
          "label": "완료 (데이터 없음)",
          "description": "수치는 없지만 계획대로 완료했어요."
        },
        {
          "label": "부분 완료",
          "description": "계획의 일부만 했어요. 어느 부분까지 했는지 알려주세요."
        }
      ],
      "multiSelect": false
    }
  ]
}
```

### Step R2: 결과 분석 [prompt]

오늘의 계획(`YYYY-WNN_plan.md`에서 오늘 행)과 실제 결과를 비교한다.

분석 항목:
- 목표 달성 여부 (거리, 페이스, 심박)
- 훈련 품질
- 주목할 신호 (통증, 피로도, 폼 변화)

출력 형식:
```
## 훈련 결과 분석 — M/D

### 계획 vs 실제
| 항목 | 계획 | 실제 |
|------|------|------|
| 거리 | N km | N km |
| 시간 | N분  | N분  |
| 페이스 | N/km | N/km |
| 심박 | — | N bpm |

### 분석
- **잘한 점**: ...
- **확인 필요**: ...
- **다음 훈련 시 반영**: ...
```

### Step R3: 파일 저장 [script]

두 곳에 저장한다.

**1. 일별 훈련 기록 생성:**
`40-training-log/daily/YYYY-MM-DD.md` — Write 도구로 신규 생성.

```markdown
# 훈련 기록 — YYYY-MM-DD (요일)

## 훈련 내용
{상세 결과}

## 분석
{Step R2 분석 내용}
```

**2. 주간 계획 파일 업데이트:**

수정 전 반드시 diff를 보여주고 확인받는다:

```
📝 훈련 기록 반영 예정:
- 파일: 40-training-log/weekly/YYYY-WNN_plan.md
- 변경 전: | M/D  | — | — |
- 변경 후: | M/D  | ✅ 완료 | {핵심 수치} |
반영할까요? [예 / 아니오]
```

확인 후 Edit 도구로 수정. `old_string`에 날짜(M/D) 포함해 행을 고유하게 지정.
- 부분 완료: `⚠️ 부분` 으로 표기
- 데이터 없는 완료: `✅ 완료 (데이터 없음)` 으로 표기
- 미수행: `❌ 미수행` 으로 표기

수정 완료 후 파일을 재읽어 변경 내용이 정확히 반영됐는지 검증한다.

### Step R4: 주간 계획 조정 검토 + 다음 훈련 안내 [prompt]

**1. 주간 계획 조정 검토**

아래 조건 중 하나라도 해당되면 남은 주간 계획 조정이 필요한지 검토한다:

- 통증 신호 발생
- 계획 대비 60% 미만 수행
- 예상보다 피로도가 높음

해당 시: 남은 주 일정에 영향을 미칠 수 있음을 알리고 `/plan-adjustment` 실행을 권유한다.
해당 없음: "이번 주 계획은 그대로 유지합니다." 한 줄로 끝낸다.

**2. 다음 훈련 미리보기**

주간 계획 파일(`YYYY-WNN_plan.md`)에서 오늘 이후 다음 훈련 세션을 찾아 한 줄로 안내한다.

- 내일 훈련이 있으면: `📅 내일 (M/D): {훈련 내용} [{소요시간}, {장소}]`
- 내일이 휴식일이고 모레 훈련이 있으면: `📅 다음 훈련 M/D (요일): {훈련 내용} [{소요시간}, {장소}]`
- 이번 주 남은 훈련이 없으면: `📅 이번 주 훈련 완료. 다음 주 계획은 /weekly-plan 으로 수립하세요.`

오늘 훈련 결과(피로도·통증)가 다음 훈련에 영향을 줄 수 있으면 한 줄 코칭 메모를 추가한다.
예) `💬 오늘 왼쪽 발목 신호 있었으니 내일 준비운동 때 발목 가동성을 충분히 확보하세요.`
