# /workout-review — 훈련 결과 심층 리뷰 & 코칭 피드백

> 계획 대비 실제 수행을 분석하고, 데이터 기반 코칭 피드백과 회복 판단을 제공한다.

---

## 핵심 원칙

- 칭찬보다 객관적 분석을 우선한다.
- 계획 대비 수행 여부를 반드시 평가한다.
- 통증 4/10 이상이면 성과 분석보다 회복 판단을 먼저 한다.
- Garmin/Strava 데이터 없이 수치만으로도 동작한다. 수치도 없으면 질적 평가만 제공.
- 리뷰 결과는 반드시 파일로 저장되어 다음 주간 계획 수립에 활용된다.

---

## 프로젝트 경로

```
WEEKLY_LOG_DIR = 40-training-log/weekly/
DAILY_LOG_DIR  = 40-training-log/daily/
PROFILE_FILE   = 20-knowledge-base/01_PROFILE.md
```

---

## 워크플로우

### Step 1: 컨텍스트 로드 [rag]

오늘 날짜로 파일 3개를 읽는다.

```python
import datetime
today = datetime.date.today()
iso = today.isocalendar()
weekly_file = f"{iso[0]}-W{iso[1]:02d}_plan.md"
daily_file  = today.strftime("%Y-%m-%d") + ".md"
```

1. `WEEKLY_LOG_DIR/YYYY-WNN_plan.md` → 오늘 계획된 세션 추출 (세션명, 거리, 강도, 목적)
2. `DAILY_LOG_DIR/YYYY-MM-DD.md` → 이미 기록된 내용 있으면 읽기 (중복 입력 방지)
3. `PROFILE_FILE` → 부상 이력, 고질적 통증 부위, 현재 목표 참조

파일이 없으면 조용히 건너뛴다 (에러 없이 진행).

---

### Step 2: 훈련 결과 입력 수집 [prompt]

AskUserQuestion 1회로 수집한다. 이미지 첨부가 가능함을 안내한다.

```json
{
  "questions": [
    {
      "question": "훈련 결과를 알려주세요. 수치가 있으면 숫자로, Garmin/Strava 화면이 있으면 이미지로 첨부해도 됩니다.",
      "header": "훈련 결과",
      "options": [
        {
          "label": "수치 직접 입력",
          "description": "예: 8km / 45분 / 페이스 5:38 / 심박 148 / 케이던스 172 / 피로도 6 / 통증 없음"
        },
        {
          "label": "Garmin/Strava 스크린샷 첨부",
          "description": "활동 요약 화면을 캡처해서 첨부해주세요. 데이터를 자동으로 읽습니다."
        },
        {
          "label": "수치 없이 완료만",
          "description": "데이터는 없지만 완료했어요. 소감과 피로도만 알려주세요."
        }
      ],
      "multiSelect": false
    }
  ]
}
```

**입력 파싱 항목:**
- 거리(km), 시간(분:초), 평균 페이스(min/km), 평균 심박(bpm), 최대 심박(bpm)
- 케이던스(spm), 고도(m), 날씨/환경 (있으면)
- 사용자 소감 (자유 텍스트)
- 피로도 (1-10), 통증 여부 + 부위 + 수준(1-10)

통증 수준이 자연어로 입력되면 ("좀 아파", "살짝") 1-10 재질문으로 수치화한다.

### Step 3: 이미지 데이터 추출 (스크린샷 첨부 시에만) [prompt]

Claude의 멀티모달 기능으로 첨부 이미지를 읽는다.

추출 항목: 거리, 시간, 평균 페이스, 평균/최대 심박, 케이던스, 고도, 랩별 페이스(있으면)

텍스트 입력 수치와 불일치 시: "이미지에서는 N, 입력하신 값은 M입니다. 어떤 값을 쓸까요?" 확인.
추출 실패 시: "이미지 추출 실패, 텍스트 수치로 진행합니다" 안내 후 계속.

---

### Step 4: 훈련 품질 분석 [generate]

**분석 우선순위 (이 순서대로):**

1. **통증 4/10 이상** → 성과 분석 중단. 회복 판단만 실행.
   - 통증 부위, 수준, 훈련 중 발생 시점 파악
   - 회복 권고 및 필요 시 병원 방문 안내
   - 계획 대비 평가 없이 "회복 우선" 결론

2. **데이터 있음** → 정량 분석 실행:
   - 계획 대비 달성률 (거리, 페이스, 시간)
   - 심박 효율: 최근 동일 페이스 세션과 심박 비교 (이력 있을 때)
   - 케이던스 평가: 목표치 180spm 기준 편차
   - 페이스 일관성: 전반/후반 또는 랩 데이터로 편차 산출

3. **데이터 없음** → 질적 평가만:
   - 사용자 소감과 피로도 기반 평가
   - "데이터 없어 정량 분석 불가" 명시

**회복 필요도 판단:**

| 조건 | 등급 | 다음 세션 조치 |
|------|------|--------------|
| 통증 4/10+ 또는 달성률 50% 미만 | 🔴 높음 | 강도 50% 이하, 회복 세션 우선 |
| 피로도 7/10+ 또는 달성률 70~80% | 🟡 보통 | 강도 유지 또는 10% 하향 |
| 달성률 80%+, 통증·피로 낮음 | 🟢 낮음 | 주간 계획대로 진행 |

### Step 6: 파일 저장 및 업데이트 [script]

저장 전 diff를 보여주고 확인받는다.

**1. Daily log 저장:**

`DAILY_LOG_DIR/YYYY-MM-DD.md` 존재 여부 확인:
- **파일 있음**: `## 훈련 리뷰` 섹션이 없으면 추가. 이미 있으면 덮어쓰기 전 확인.
- **파일 없음**: 아래 형식으로 신규 생성.

```markdown
# 훈련 기록 — YYYY-MM-DD (요일)

## 훈련 내용
{거리, 시간, 페이스, 심박, 케이던스, 피로도, 통증, 소감}

## 훈련 리뷰
{Step 5 출력 내용}
```

**2. 주간 계획 훈련 기록 표 업데이트:**

`YYYY-WNN_plan.md`의 해당 날짜 행을 수정한다:
- `old_string`에 반드시 날짜(M/D) 포함해 행 고유성 확보
- `— | —` → `✅ 완료 | {핵심 수치 한 줄}` 또는 `⚠️ 부분 | ...` 또는 `❌ 미수행 | ...`
- 수정 후 파일 재읽어 반영 확인

**3. 주간 계획 수정 필요 시:**
사용자에게 알리고 `/weekly-plan` 재실행 권유. 자동 수정 안 함.

## 폴백 처리

| 상황 | 처리 |
|------|------|
| 주간 계획 파일 없음 | 계획 비교 없이 결과만 분석, "주간 계획 없음" 표시 |
| 데이터 수치 없음 | 질적 평가만, "데이터 없음" 명시, 저장은 진행 |
| 이미지 추출 실패 | 텍스트 수치로 진행, 실패 사실 안내 |
| 통증 4/10 이상 | 회복 최우선, 성과 분석 스킵, 필요 시 병원 방문 안내 |
| 파일 저장 실패 | 대화에 결과 유지, 에러 원인 안내 |
